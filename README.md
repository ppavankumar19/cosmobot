# CosmoBot

Single-page space exploration chatbot built with vanilla HTML/CSS/JS, a local Three.js background shader, a Groq-backed chat proxy, and a Sarvam-backed text-to-speech proxy.

**Live site:** https://cosmobot.19062002.xyz/
**Entire.io project:** https://entire.io/gh/ppavankumar19/cosmobot/checkpoints/main

---

## What The App Does

- Space-focused AI chat powered by Groq (`llama-3.3-70b-versatile`)
- Animated splash screen with shuffle-text title (`COSMOBOT`)
- Fullscreen Color Bends WebGL background with pointer-responsive parallax
- Welcome view with rotating space fact and 6 suggestion chips
- Markdown-rendered bot replies (bold, lists, code blocks, links, blockquotes)
- Copy and Sarvam-powered "Listen" (TTS) actions on every bot message
- Typing indicator, inline error bubbles, timestamps, character counter, scroll-to-bottom FAB
- Fully responsive ‚Äî mobile, tablet, desktop

## Current Stack

| Layer | Implementation |
|-------|----------------|
| Frontend | `public/index.html` ‚Äî inline CSS + inline module script, no build step |
| Background | ReactBits-inspired Color Bends shader via Three.js v0.183 |
| Markdown | `marked` v12 + `DOMPurify` v3 from jsDelivr CDN |
| Chat API | `api/chat.js` ‚Üí Groq `llama-3.3-70b-versatile` |
| TTS API | `api/tts.js` ‚Üí Sarvam `bulbul:v3`, speaker `manan`, `en-IN` |
| Local dev server | `server.js` ‚Äî Express on `http://localhost:8555` |
| Deployment | Vercel (serverless functions + static assets) |

## Local Setup

1. Install dependencies (also copies Three.js bundles into `public/`):

```bash
npm install
```

2. Copy `.env.example` to `.env` and fill in your API keys:

```bash
cp .env.example .env
# Edit .env with your actual keys
```

3. Start the local dev server:

```bash
npm run local
```

4. Open in your browser:

```text
http://localhost:8555
```

For Vercel deployments, configure both `GROQ_API_KEY` and `SARVAM_API_KEY` in the Vercel project environment settings.

## API Reference

### `POST /api/chat`

**Request:**
```json
{
  "messages": [
    { "role": "user", "content": "What is the Artemis program?" }
  ]
}
```

**Response (success):**
```json
{
  "message": "The Artemis program is NASA's initiative to..."
}
```

**Response (error):**
```json
{
  "error": "..."
}
```

The handler prepends a server-owned CosmoBot system prompt and sends the last 20 messages to Groq using `llama-3.3-70b-versatile` (`temperature: 0.72`, `max_tokens: 1024`, `stream: false`).

---

### `POST /api/tts`

**Request:**
```json
{
  "text": "Explain how the James Webb Space Telescope works."
}
```

**Response:** Raw MP3 audio bytes (`Content-Type: audio/mpeg`). The frontend creates a blob URL and plays it via an `Audio` element.

Current Sarvam voice settings (fixed server-side):

| Setting | Value |
|---------|-------|
| `model` | `bulbul:v3` |
| `speaker` | `manan` |
| `target_language_code` | `en-IN` |
| `pace` | `1.1` |
| `speech_sample_rate` | `22050` |
| `output_audio_codec` | `mp3` |

The handler proxies Sarvam server-side ‚Äî the browser never sees the `SARVAM_API_KEY`.

## Implementation Notes

- The app is intentionally build-free on the frontend. No bundler, no transpiler.
- Three.js is loaded via a local import map (`/three.module.min.js`) copied by `postinstall`.
- All background effects, splash animation, chat UI logic, and accessibility hooks live in `public/index.html`.
- Chat responses come from Groq; audio comes from Sarvam and plays in-browser via a streamed blob.
- See `PROCESS_FLOW.md` for the end-to-end data flow and `WORKFLOW.md` for development and deployment steps.
